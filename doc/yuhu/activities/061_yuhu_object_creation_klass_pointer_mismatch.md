## Executive Summary

Yuhu compiler creates objects with **full 64-bit klass pointers**, but RuntimeStub generated by C1 assumes **compressed klass pointers** when `UseCompressedClassPointers` is enabled. This mismatch causes incorrect klass decoding and crashes during finalizer registration.

**Root Cause**: Inconsistent klass pointer encoding between Yuhu object creation and RuntimeStub klass access.

**Evidence**: 
```assembly
# RuntimeStub generated code (assumes compressed klass)
ldr    w5, [x0, #0x8]      # Read klass as 32-bit compressed oop
lsl    x5, x5, #3          # Decode: x5 = compressed << 3
ldr    w5, [x5, #0xa4]     # ❌ CRASH: Invalid address

# Actual object memory (full 64-bit klass pointer)
0x76ab1dba8: 0x0000000000000005  0x00000000f800612b
#                              ^-- Full 64-bit klass pointer (not compressed)
```

## Root Cause Analysis

### The Encoding Mismatch

**Yuhu creates objects**:
```
Object at: 0x000000076ab1dba8
Memory:    0x0000000000000005  0x00000000f800612b
                              ^-- klass field = 0x00000000f800612b (full pointer)
```

**RuntimeStub decodes**:
```
Step 1: ldr w5, [x0, #0x8]  → w5 = 0xf800612b (low 32 bits)
Step 2: lsl x5, x5, #3      → x5 = 0x7C0030958 (decoded)
Step 3: ldr w5, [x5, #0xa4] → CRASH (0x7C0030958 is not valid klass)
```

**Actual klass address**: `0x00000007c000d480` (from lldb verification)

**Decoded address**: `0x00000007c0030958` (wrong!)

**Difference**: `0x7c000d480 ≠ 0x7c0030958`

### Why This Happens

**JVM Configuration**:
- `UseCompressedClassPointers = true` (enabled)
- `narrow_klass_base` and `narrow_klass_shift` configured
- RuntimeStub generated with compressed klass assumption

**Yuhu Behavior**:
- `CreateInlineMetadata` embeds **full 64-bit pointer** (not compressed)
- Ignores `UseCompressedClassPointers` configuration
- Stores `0x00000007c000d480` directly to object's klass field

**Result**: 
- Object has full klass pointer
- RuntimeStub assumes compressed klass
- Decoding produces wrong address → crash

### Compressed Klass Encoding

**If klass were properly compressed**:
```
compressed_klass = (full_klass - narrow_klass_base) >> narrow_klass_shift
```

**Yuhu stores** (incorrect):
```
klass_field = full_klass  // 0x00000007c000d480
```

**Should store** (if compressed):
```
klass_field = compressed_klass  // e.g., 0x0006a480 (32-bit)
```

## Java Object Creation Flow

### Bytecode Level

```java
StringBuilder sb = new StringBuilder();
```

**Compiled bytecode**:
```
0: new           #2     // Allocate object, initialize klass
3: dup                  // Duplicate reference
4: invokespecial #3     // Call <init>(this)
7: astore_1             // Store to local variable
```

### Execution Steps

**Step 1: `new` instruction** (object creation)
```
1. Allocate memory (TLAB or heap)
2. Zero out memory
3. Set mark word
4. Set klass pointer ← Yuhu code (yuhuTopLevelBlock.cpp:2033-2034)
5. Push object reference to stack
```

**Step 2: `dup` instruction**
```
Stack: [object_ref, object_ref]
```

**Step 3: `invokespecial <init>`**
```
Stack: [object_ref]  ← Pass as 'this' parameter
Call: StringBuilder.<init>(this)
```

**Key Point**: Object is **fully created** (with klass set) **before** `<init>` is called!

### In Our Crash Scenario

```
1. ✅ new StringBuilder → Yuhu creates object, sets klass = 0x7c000d480 (full pointer)
2. ✅ Call StringBuilder.<init>()
3. ✅ StringBuilder.<init>() calls Object.<init>()
4. ✅ Object.<init>() return → call_register_finalizer(receiver)
5. ❌ RuntimeStub tries to decode klass (assumes compressed) → CRASH
```

**Key Implementation Details**:

1. **`encode_klass_not_null`**:
   - Input: `ptr` type (full klass pointer from `CreateInlineMetadata`)
   - Convert: `ptr` → `i64` using `CreatePtrToInt`

2. **`decode_klass_not_null`**:
   - Input: `i32` type (compressed klass from memory load)
   - Extend: `i32` → `i64` using `CreateZExt`
   - Decode: Perform arithmetic on `i64` (shift, addition)
   - Output: `ptr` type (full klass pointer for comparison)

3. **LLVM IR Type Safety**:
   - Arithmetic operations (`lshr`, `shl`, `add`, `sub`) require integer types
   - Cannot use `ptr` type with arithmetic instructions
   - Must convert: `ptr` ↔ `i64` using `ptrtoint`/`inttoptr`

## Key Takeaways

### 1. Object Creation Precedes `<init>`

Java's `new` instruction **creates the object first** (including setting klass), then calls `<init>` to initialize fields. The object is fully formed before constructor runs.

### 2. Klass Pointer Encoding Must Be Consistent

All code that creates or accesses objects must use the same klass encoding (compressed vs full). Mixing encodings causes crashes.

### 3. RuntimeStub Configuration Matters

RuntimeStub is generated once at JVM startup based on `UseCompressedClassPointers`. All compiled code must respect this configuration.

### 4. Compressed OOPs Are Not Optional

When JVM enables compressed oops/klass, **all** code must use compressed encoding. Cannot mix full pointers with compressed assumptions.

## Solution (Decided)

### Approach: Conditional Klass Pointer Storage

**Principle**: Store klass pointer based on `UseCompressedClassPointers` configuration.

**Implementation**:
```cpp
if (UseCompressedClassPointers) {
  // Encode and store compressed klass (32-bit)
  Value *compressed = encode_klass_not_null(full_klass);
  builder()->CreateStore(compressed, klass_addr_32);
} else {
  // Store full klass pointer (64-bit)
  builder()->CreateStore(full_klass, klass_addr_64);
}
```

**Rationale**:
- ✅ Consistent with C1/C2 implementation (macroAssembler_aarch64.cpp:3187-3196)
- ✅ Matches RuntimeStub expectations
- ✅ Supports both compressed and uncompressed modes
- ✅ Memory efficient (4 bytes vs 8 bytes per object in compressed mode)

### Implementation Scope

**Must modify**:
1. ✅ **Object creation** - Store klass with proper encoding
2. ✅ **Type checking** - Load and decode klass from object (instanceof, checkcast)
3. ✅ **Virtual method calls** - Load and decode klass for vtable lookup
4. ✅ **Exception handling** - Load and decode klass for catch block matching

**Reason**: All code that reads object's klass field must decode if compressed, all code that writes must encode if compressed.
